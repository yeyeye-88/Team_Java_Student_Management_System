# 学生管理系统 - Java课设项目

## 项目概述

本项目为Java课程设计四人协作开发的学生管理系统，遵循**8周精细化开发流程**，以冲刺课程设计90+高分为目标，严格贴合课程考核要求完成开发。系统实现了用户权限管理、学生/课程/成绩/考勤基础信息维护、请假审批复杂业务流程、数据统计可视化等核心功能，技术栈与开发规范完全遵循课程指定要求，全程采用**1:1:1:1均等分工原则**，保障团队成员工作量一致，最终实现项目高质量交付。

## 核心技术栈

严格遵循课程指定技术栈，无额外框架/工具引入：

### 阶段2：功能深化期（第4-6周）| 提高分≥80分

1. 扩展**课程、成绩、考勤**3个基础信息模块，实现全模块增删改查与数据关联；

### 阶段3：冲刺提分期（第7-8周）| 冲刺分≥90+分

1. 实现**4类数据统计可视化**：班级学生数（柱状图）、课程平均分（折线图）、考勤率（饼图）、请假通过率（环形图）；

4. 完成**线上部署**，按课程要求整理并上传所有提交资料。

## 项目核心功能模块

### 1. 用户/权限管理模块

- 3角色账号登录：管理员（admin/123456）、学生（stu01/123456）、教师（tea01/123456）；

- 细粒度权限控制：不同角色可见菜单/操作按钮不同，接口级权限校验；

- 角色增删改：管理员可维护角色账号信息。

### 2. 基础信息管理模块

- 学生管理：基本信息增删改查、按学号/姓名/班级模糊查询；

- 课程管理：课程信息维护、按课程名查询、关联学生选课信息；

- 成绩管理：成绩录入/修改/删除、按学生/课程查询、成绩范围校验（0-100）；

- 考勤管理：考勤状态（正常/迟到/缺勤/请假）维护、关联请假流程同步状态。

### 3. 请假审批模块

- 学生端：请假信息提交（类型/时长/原因）、表单校验、个人请假记录查询；

- 管理员端：请假记录审核（通过/驳回）、按学生/审批状态筛选、审批结果同步考勤；

- 流程优化：防重提交、状态流转控制、操作结果实时反馈。

### 4. 数据统计可视化模块

- 班级学生数统计：按班级分组统计学生数量，柱状图展示；

- 课程平均分统计：按课程分组计算平均分，折线图展示；

- 考勤率统计：统计各考勤状态占比，饼图展示；

- 请假通过率统计：统计请假通过/驳回占比，环形图展示；

- 交互功能：图表动态切换、数据详情展示、图表导出为图片。

## 项目工程结构

### 后端工程：java-server

遵循课程指定包结构，所有类/方法命名符合Java驼峰规范：

```Plain Text

java-server/
├── src/main/java/
│   ├── cn/edu/sdu/java/sever/        # 团队包名
│   │   ├── configs/     # 配置类（Spring Security、JWT、数据库等）
│   │   ├── models/      # JPA实体类（与数据库表一一对应）
│   │   ├── repositorys/ # JPA数据访问层接口（继承JpaRepository）
│   │   ├── services/    # 业务逻辑层
│   │   ├── controllers/ # 接口层（RESTful接口）
│   │   ├── payload/     # 请求/返回数据封装类
│   │   ├── util/        # 通用工具类（参数校验、角色判断、响应封装等）
│   │   └── JavaServerApplication.java # 后端主启动类
│   └── resources/
│       ├── application.yml # 核心配置文件（端口、数据库、JWT等）
│       └── java-db.sql     # 基础数据库SQL文件
└── pom.xml # Maven依赖配置文件
```

### 前端工程：java-fx

遵循课程指定规范，FXML按模块命名，Controller与FXML一一对应：

```Plain Text

java-fx/
├── src/main/java/
│   ├── cn/edu/sdu/java/sever/        # 团队包名
│   │   ├── controller/  # 前端控制器（处理界面交互、接口调用）
│   │   ├── util/        # 前端工具类（数据渲染、交互提示、HTTP请求等）
│   │   ├── AppStore/    # 课程基础类（状态管理）
│   │   └── MainApplication.java # 前端主启动类
│   └── resources/
│       ├── fxml/        # FXML界面文件（按模块分类：login、student、course等）
│       ├── css/         # 统一样式文件（界面风格统一）
│       └── images/      # 静态资源（logo、团队信息等）
└── pom.xml # Maven依赖配置文件
```

## 项目运行与部署

### 本地运行步骤

1. **数据库初始化**

    - 用MySQL Workbench连接团队数据库服务器（地址：[202.194.14.120](202.194.14.120)，端口按班级分配）；

    - 导入项目根目录下`sql/`文件夹中的全量SQL文件，包含基础表、扩展表、测试数据；

    - 验证数据库表结构、约束、测试数据是否正常。

2. **后端启动**

    - 用IDEA打开`java-server`工程，执行Maven `Reload Project`同步依赖；

    - 修改`resources/application.yml`中的数据库连接信息（与团队数据库一致）；

    - 运行`JavaServerApplication.java`主类，或直接运行打包后的jar包：`java -jar java-server-1.0-SNAPSHOT.jar`；

    - 验证后端启动成功：PostMan调用`/api/login`接口，可正常返回JWT token。

3. **前端启动**

    - 用IDEA打开`java-fx`工程，执行Maven `Reload Project`同步依赖；

    - 修改`util/HttpRequestUtil.java`中的`serverUrl`，对接后端服务端口；

    - 运行`MainApplication.java`主类，或直接解压前端zip包后运行可执行文件；

    - 用测试账号登录，验证各功能模块正常使用。

### 线上部署步骤

1. 后端：将打包后的`java-server-1.0-SNAPSHOT.jar`上传至线上服务器，执行`nohup java -jar xxx.jar &`后台启动；

2. 前端：将前端zip包上传至线上服务器，解压后直接运行可执行文件；

3. 防火墙配置：开放后端服务端口与数据库端口，确保前后端、前端与数据库网络互通；

4. 功能验证：线上访问系统，测试所有核心功能，确保运行稳定。
